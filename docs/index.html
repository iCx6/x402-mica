§ 1 — Compliance

Payments are the easy part. The paperwork isn't.

The x402 stack handles the 402 challenge and on-chain settlement. What EU-facing sellers additionally need — a record of who paid what, in which asset, settled where — is what this library adds on top.

I. custody

Non-custodial by design

Funds move payer-wallet → your wallet through the x402 facilitator. This library never touches, holds, or forwards money — it observes and records. That's what keeps you a software vendor, not a payment institution.

II. audit

MiCA audit trail, built in

Every settled request logs timestamp, asset, amount, payer address, the facilitator's transaction reference, and a mica_compliant flag to SQLite. A read-only, key-gated dashboard exports it as HTML, CSV, or JSON, filtered by date range.

III. assets

USDC & EURC on Base

USDC by default; Circle's euro stablecoin EURC as a one-line opt-in (asset: "EURC"). Both are e-money tokens issued under Circle's French EMI license — MiCA-compliant assets, priced in dollars or euros as you prefer.

§ 2 — Integration

Two integration points, one line each

Paywall an Express route, or wrap any MCP tool so AI agents pay per call — same options, same audit trail, transport-agnostic.

Exhibit A — Express middleware

import express from "express";
import { x402Middleware } from "x402-mica";

const app = express();

app.use(x402Middleware({
  route: "GET /demo",
  price: "$0.01",
  asset: "USDC",          // or "EURC" with price "0.01"
  network: "eip155:84532", // Base Sepolia; "eip155:8453" = mainnet
  payTo: "0xYourWallet",
  dbPath: "./audit.db",
}));

app.get("/demo", (_req, res) => res.json({ ok: true }));
app.listen(3000);

Exhibit B — MCP tool decorator

import { withPayment } from "x402-mica";

server.tool("echo", "Requires $0.01 USDC.",
  { text: z.string() },
  withPayment(async ({ text }) => ({
    content: [{ type: "text", text }] }), {
    price: "$0.01", asset: "USDC",
    network: "eip155:84532",
    payTo: "0xYourWallet",
    dbPath: "./audit.db",
  }),
);
// Transport-agnostic: stdio + Streamable HTTP —
// the payment travels inside MCP.

Exhibit C — Audit dashboard

import { auditDashboard } from "x402-mica";

app.get("/audit", auditDashboard({
  dbPath: "./audit.db",
  apiKey: process.env.AUDIT_API_KEY,
}));

// Read-only, key-gated. Export the log:
//   /audit?format=csv&from=2026-04-01&to=2026-06-30
//   /audit?format=json

Unpaid requests get an HTTP 402 challenge with payment instructions; paid requests pass through and are audit-logged. Verification and settlement are delegated to the facilitator — no hand-rolled signature checks, no custody.
